================================================================================
FORMATTING CONVENTIONS
================================================================================

Character Set for Tables:
  ┌ ─ ┬ ┐    Top corners and horizontal
  │   │ │    Vertical separators
  ├ ─ ┼ ┤    Row separators with intersections
  └ ─ ┴ ┘    Bottom corners

Status Values:
  Pass       - Principle fully satisfied (or confidence > 60%)
  Warning    - Principle partially satisfied (confidence 30-60%)
  Fail       - Principle not satisfied (confidence < 30%)
  N/A        - Principle not applicable to this session

Confidence Display:
  [##%]      - Single value when available
  [##-##%]   - Range when multiple principles aggregated

Dividers:
  ---        Section separator (three dashes)

Theme Headers:
  ALL CAPS   - Theme names in principle-level detail section

================================================================================
USAGE GUIDELINES
================================================================================

When to Use This Template:
- After running validation tools against a session plan
- To compare findings across multiple theoretical frameworks
- Before creating an improvement plan (this identifies what to improve)

Document Flow:
  1. Evaluation Comparison (this template) - Identifies gaps
  2. Improvement Plan (improvement-plan.txt) - Addresses gaps

Key Sections:
1. Overall Assessment - Quick summary for busy readers
2. Side-by-Side - Framework-specific detail
3. Principle-Level Detail - Thematic grouping shows convergence
4. Strength & Gap Matrix - Visual overview of session elements
5. Unified Recommendations - Prioritized action items
6. Summary - Key takeaways and core insight

Thematic Groupings:
- Group related principles from different frameworks together
- Use consistent theme names across evaluations for comparison
- Identify "Convergent Gap" when all frameworks flag same issue
- Identify "Partial Strength" when most frameworks pass

Prioritization Logic:
- All frameworks agree = Priority 1-3 (High impact)
- Most frameworks agree = Priority 4-6 (Medium impact)
- Single framework flags = Priority 7+ (Low-Medium impact)
